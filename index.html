OOP Implementation

Object-Oriented Concepts

Every major OOP pillar is implemented throughout the system, demonstrating comprehensive understanding of Java and software engineering principles.

01

Encapsulation

Private fields with public getters/setters in Student, Teacher, and Course classes ensuring data hiding and controlled access.

PILLAR 1
02

Inheritance

Person → Student, Teacher, Staff class hierarchy. AdminUser extends User class with extended permissions and role-specific behaviors.

PILLAR 2
03

Abstraction

Abstract base classes and interfaces defining contracts. IManageable, IReportable interfaces used across all major system modules.

PILLAR 3
04

Polymorphism

Runtime polymorphism via method overriding. A single reference type handles different user types — Admin, Teacher, Student seamlessly.

PILLAR 4
05

Overloading

Method overloading used in search(), addRecord(), and generateReport() methods allowing multiple input signatures for flexibility.

COMPILE-TIME
06

Overriding

toString(), displayInfo(), calculateGPA() methods overridden in subclasses providing class-specific implementations of inherited methods.

RUNTIME
07

Exception Handling

Custom exceptions: StudentNotFoundException, InvalidGradeException. Try-catch-finally blocks throughout ensuring robust error recovery.

ROBUSTNESS
08

File Handling

BufferedReader, FileWriter, ObjectInputStream for serializing and deserializing Java objects. Student records persisted to .dat and .txt files.

PERSISTENCE

Architecture

Working Architecture

Layered architecture following separation of concerns — from user interaction all the way to persistent storage.

LAYER 1

User Interface

Admin, Teacher & Student access via CLI menus and web portal

LAYER 2

Authentication Layer

Login validation, role verification, session management

LAYER 2

Access Control

Role-based permissions and module access restrictions

LAYER 3

Business Logic

Java classes handling core operations per module

LAYER 3

OOP Classes

Student, Teacher, Course, Attendance, Result objects

LAYER 3

Service Layer

CRUD operations, GPA calculations, report generation

LAYER 4

Data Persistence

File I/O — .dat, .txt, serialized Java objects

LAYER 4

Report Engine

Formatted output files for results and attendance

OUTPUT

System Output

Reports, Grade Cards, Attendance Sheets, Console Displays
